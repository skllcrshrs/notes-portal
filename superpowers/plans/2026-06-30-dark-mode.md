# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a light/dark mode toggle to the Kiln header with a soft dark palette, using CSS custom properties so both schemes share one set of element rules.

**Architecture:** MkDocs Material's built-in palette toggle handles scheme switching and localStorage persistence. `mkdocs.yml` gets a two-entry palette list; `extra.css` is refactored so all element rules reference `--color-*` variables defined in `:root` (light) and `[data-md-color-scheme="slate"]` (dark).

**Tech Stack:** MkDocs Material theme, CSS custom properties, `mkdocs serve` for local verification

## Global Constraints

- Only `mkdocs.yml` and `docs/stylesheets/extra.css` change — no other files are touched
- No JavaScript changes required
- Dark palette: bg `#1e2028`, surface `#252830`, border `#2e3138` (soft dark, not true black)
- Accent blue `#2563eb` is identical in both schemes
- Font, layout, and spacing rules are scheme-agnostic — only color values change

### Task 1: mkdocs.yml — palette toggle

**Files:**
- Modify: `mkdocs.yml`

**Interfaces:**
- Produces: Material palette toggle rendered in header; `[data-md-color-scheme]` attribute set on `<body>` that CSS targets in later tasks

- [ ] **Step 1: Replace the single palette block with a two-entry list**

In `mkdocs.yml`, find:
```yaml
  palette:
    scheme: default
    primary: white
    accent: blue
```

Replace with:
```yaml
  palette:
    - scheme: default
      primary: white
      accent: blue
      toggle:
        icon: material/weather-night
        name: Switch to dark mode
    - scheme: slate
      primary: black
      accent: blue
      toggle:
        icon: material/weather-sunny
        name: Switch to light mode
```

- [ ] **Step 2: Verify the toggle renders**

Run: `mkdocs serve`
Open: `http://127.0.0.1:8000`

Expected: A moon icon appears in the top-right of the header. Clicking it switches to dark mode and shows a sun icon. Reloading the page keeps whichever mode was selected (localStorage persistence). At this stage dark mode will look visually broken (hardcoded light colors still in CSS) — that is expected and will be fixed in Tasks 2–4.

- [ ] **Step 3: Commit**

```bash
git add mkdocs.yml
git commit -m "feat: add dark mode palette toggle to mkdocs.yml"
```
